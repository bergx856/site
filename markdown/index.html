Markdown Site Building

The Markdown rundown! These are the steps that I follow to generate a live preview of my site, as well as those that I use to turn that preview into a live and accessible site via GitHub.

Testing

Command prompt lines used to create a live, re-loading docs server:

C:\Users\sara's pc>cd desktop
C:\Users\sara's pc\Desktop>cd mkproj1
C:\Users\sara's pc\Desktop\mkproj1>cd project
C:\Users\sara's pc\Desktop\mkproj1\project>mkdocs serve

These four lines are simply telling the computer where to find the specific folder of Markdown files that I would like to access and use to generate my live site. My folders are located on my desktop in a folder entitled "mkproj1", which has the final destination file entitled "project". Inputting that final "mkdocs serve" then generates:

INFO    -  Cleaning site directory
INFO    -  Documentation built in 0.09 seconds
[I 210427 19:35:45 server:335] Serving on http://127.0.0.1:8000
INFO    -  Serving on http://127.0.0.1:8000
[I 210427 19:35:45 handlers:62] Start watching changes
INFO    -  Start watching changes
[I 210427 19:35:45 handlers:64] Start detecting changes
INFO    -  Start detecting changes
[I 210427 19:35:49 handlers:135] Browser Connected: http://127.0.0.1:8000/conception/
INFO    -  Browser Connected: http://127.0.0.1:8000/conception/

This handy set of lines allows me to easily see what happens when I make a change on my text editor Atom. I simply save my change in Atom, and it will update instantly to to the preview of my site! This site can then be tested at the provided link: http://127.0.0.1:8000.

Launching

Command prompt lines used to build the documentation site:

C:\Users\sara's pc>cd desktop
C:\Users\sara's pc\Desktop>cd mkproj1
C:\Users\sara's pc\Desktop\mkproj1>cd project
C:\Users\sara's pc\Desktop\mkproj1\project>mkdocs build

These four lines are nearly identical to the first four used to create the live server, the only difference being the command "build" instead of "serve" in the last line. The "build" creates a site folder within my project folder, which can then be published by using GitHub. On Github desktop, I click "file", "add local repository", choose the site folder as the destination, and hit add repository. I keep the repository's name as "site" for ease, and initialize with a ReadMe. Then, my site is officially live!

To update new changes to my actual site after this initial draft, I simply redo the "mkdocs build" command, which updates my site folder. This then brings up all of the changes that I've made into Github. After writing a short summary and description of the changes that I've made, I hit commit to main, and push origin. The site is now live with my most recent updates!
